# ADR: Drizzle ORM instead of TypeORM

**Status:** Accepted · **Date:** 2026-05-19

## Context

The challenge brief lists TypeORM as the obligatory ORM. The target runtime is Next.js 16 App Router on Vercel — a serverless platform with cold-start cost and no long-lived connections.

## Problem

TypeORM was designed for long-running Node processes:
- It expects a singleton `DataSource` that connects once and pools persistent connections.
- It uses decorators + `reflect-metadata`, both of which fight Turbopack and require special tsconfig flags.
- Entity classes carry both runtime behavior and schema metadata; serverless cold starts pay that re-init cost on every invocation.
- Migrations are a CLI separate from the runtime entities; configuring it across environments is non-trivial.

Drizzle was designed in the opposite direction:
- Plain functions + plain objects; no decorators.
- One stateless connection per invocation, friendly to pgBouncer transaction mode (which Supabase exposes on port 6543).
- Schema is regular TypeScript; types are inferred, not declared twice.
- Migrations are SQL files generated by `drizzle-kit`, version-controlled and reviewable.

## Decision

Use Drizzle ORM. Document the deviation from the brief.

## Alternatives considered

| Option | Why we didn't pick it |
|---|---|
| TypeORM (brief default) | Above. |
| Prisma | Heavier (separate query engine binary), still has cold-start cost on serverless, and `prisma generate` adds a moving part to CI. Not a deal-breaker, just not Drizzle's level of simplicity. |
| Supabase client only (no ORM) | We'd lose typed joins and have to hand-write SQL for things Drizzle does for free. Acceptable but a step backwards in DX for a 3-day MVP. |
| Kysely | Comparable to Drizzle; we don't have a reason to prefer it and the wider ecosystem around Drizzle is currently larger. |

## Consequences

**Positive.**
- Edge-friendly database access pattern.
- Migrations are plain SQL — easy to review and rollback.
- Type inference flows from schema definitions, no codegen step.

**Negative.**
- Deviates from the brief; we own the burden of justifying the choice (this ADR).
- Smaller ecosystem of third-party plugins compared to TypeORM. Not material for this app.

## How to revisit

If the evaluator requests TypeORM strictly, the schema in `packages/db/src/schema/` is small enough to port in a few hours — entities map 1:1 to current Drizzle tables, and the migration SQL can be ported by hand.
